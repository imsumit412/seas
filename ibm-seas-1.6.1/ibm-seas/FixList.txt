PLEASE READ: Before starting the SEAS installation, upgrade, or rollback process, please follow the steps outlined below:

================================================================================
----- Setting Permissions on Storage -----
Before deploying the Helm chart, review the following section in the values.yaml file:
# rootSquashCheck: This flag indicates whether the deployment user has reviewed the rootSquash requirements.
# Set it to true if the deployment user has reviewed the rootSquash requirements.
# Set it to false if the deployment user has not reviewed the rootSquash requirements.
rootSquashCheck: false

# rootSquash Settings:
rootSquash:
    # Set the enabled flag to true if root squash is enabled on the mounted PV server.
    # set it to false - If root squash is disabled on the mounted PV server. In this scenario, the umask should be set to 002, since the mounted directory is owned by the root user and the group needs write permissions. Additionally, the mounted PV directory should have 775 permissions  
    enabled: false
    # It should correspond to either the fsGroup or supplementalGroup ID 
    runAsUser: 65534

	
----- Fresh install in OpenShift -----
Run Post-delete scripts:::: 
Remove the existing security context constraints if an older version of the SCC (Helm chart version < 1.5.2) is present in the cluster, run the Post-delete scripts as a cluster admin after extracting the downloaded package

Post-delete script for cluster
# ibm-seas/ibm_cloud_pak/pak_extensions/post-delete/clusterAdministration/deleteSecurityClusterPrereqs.sh

Post-delete script for namespace
# ibm-seas/ibm_cloud_pak/pak_extensions/post-delete/namespaceAdministration/deleteSecurityNamespacePrereqs.sh <Namespace/Project>

Run Pre-install scripts::::
If this is your first installation or you've removed a previous version of SCC, run the pre-installation scripts as a cluster admin after extracting the downloaded package. These scripts will configure the necessary security contexts

Pre-install script for cluster
# ibm-seas/ibm_cloud_pak/pak_extensions/pre-install/clusterAdministration/createSecurityClusterPrereqs.sh

Pre-install script for namespace
# ibm-seas/ibm_cloud_pak/pak_extensions/pre-install/namespaceAdministration/createSecurityNamespacePrereqs.sh <Namespace/Project>


----- Upgrade in OpenShift -----
Run Post-delete scripts:::: 
Remove the existing security context constraints If you are upgrading from an older version (Helm chart version < 1.5.2), run the Post-delete scripts as a cluster admin after extracting the downloaded package

Post-delete script for cluster
# ibm-seas/ibm_cloud_pak/pak_extensions/post-delete/clusterAdministration/deleteSecurityClusterPrereqs.sh

Post-delete script for namespace
# ibm-seas/ibm_cloud_pak/pak_extensions/post-delete/namespaceAdministration/deleteSecurityNamespacePrereqs.sh <Namespace/Project>

Run Pre-install scripts::::
Execute the pre-install scripts to set up security contexts for the new version.

Pre-install script for cluster (as a cluster admin)
# ibm-seas/ibm_cloud_pak/pak_extensions/pre-install/clusterAdministration/createSecurityClusterPrereqs.sh

Pre-install script for namespace (as a cluster admin)
# ibm-seas/ibm_cloud_pak/pak_extensions/pre-install/namespaceAdministration/createSecurityNamespacePrereqs.sh <Namespace/Project>


----- Rollback in OpenShift -----
Please note that downgrading from version V1.5.2 to an earlier version is not supported because of changes to the Security Context.
